## 1. 你最关心的 3 个问题（直接回答）

### Q1：Dashboard 数据从哪来？是数据库吗？

**不是数据库。两个 jsonl 文件 + 内存 Map 聚合。**

| 数据源 | 角色 | 当前规模 |
|---|---|---|
| `_meta/live-events.jsonl` | 实时事件流（hook 写入，SSE 推前端） | 705+ 行 / 1.78 MB |
| `_meta/helix-runs.jsonl` | helix run 历史（每次 start/phase/finalize 都追加） | 86 行 |
| `_meta/.helix-current-run.json` | 当前 active run state（只一份） | 单文件 |
| 内存 `sessions: Map<sid, Session>` | 由 jsonl 启动 + tail 重建 | 17 sessions |
| 内存 `tasks: Map<tid, Task>` | 同上 | 78 tasks |

**Dashboard 自报告**（views.js:676）：`Node stdlib · no SQLite · no external deps`。

之前页脚显示的 `SQLite 3.45.1 + WAL · 4 workers · 8.2k events/min` 是 **完全 hardcoded 的假文案**（app.js:51-53），本次 v0.7 已删，改为读 `/api/health` 实时数据。

### Q2：会自动记忆所有 helix 执行历史吗？

**会。每次 /helix 全程记账。**

- `--start` → 写一条 `type:start`
- 每个 phase 的 `--report` → 写一条 `type:phase_report`（含 passes/summary/duration_ms/score 等）
- `--finalize` → 写一条 `type:finalize`（含 promise + warnings）
- 永久保存，重启 dashboard 不丢

v0.7 新加 **月度轮转**：`node _meta/rotate.cjs --month YYYY-M` 把上月行归档到 `_meta/archive/helix-runs-YYYY-M.jsonl`，主文件保持小（不会膨胀到 GB）。

### Q3：历史会话能看吗？

**这次 Worker A 续作 2 已经完整加了 Sessions 视图。**

- 左侧 nav 新增 `▣ Sessions`（位置：Live 下方、Insights 上方）
- 17 个历史会话全部可见
- 每个 session card：
  - 折叠态：短 id（前 8 字符 mono 字体）+ LIVE/ended Tag + 时长 + 任务统计
  - 展开态：top 10 工具柱状图 + 该 session 的 helix runs 列表 + 散落事件流
- 三段式过滤（全部/今日/本周）+ session_id 实时搜索 + 20s 周期刷新
- 懒加载（首次展开才拉 timeline，缓存）

API 后端：`GET /api/sessions/:session_id/timeline` 返回 `{short_id, started_at, duration_minutes, status, event_count, tool_usage{}, helix_runs[], loose_events[]}`。
