# A5 executor — 执行元设计草案

> **状态**: v0.1（2026-5-1）
> **优先级**: P6（Claude 已在做，正式化即可）
> **职责**: 按 PlanDoc 执行代码修改，一步一验证，遇到风险调用 A8

## 执行原则

1. **最小变更**：只改 PlanDoc 指定的文件，不顺手改周边
2. **一步一确认**：每完成一个文件，更新进度，不批量静默修改
3. **遇到破坏性操作**：自动 invoke A8 risk-guard，不自行判断
4. **失败即停**：遇到无法解决的问题，停下来报告，不猜测前进

## 执行流程

```
读 PlanDoc
  → 确认改动文件列表
  → 逐文件修改（Edit 工具）
  → 每步完成后输出 "✅ 完成：src/auth.ts"
  → 全部完成 → 调用 A6 validator
  → 若 A6 失败 → 分析原因 → 修复 → 重新 validate
```

## 输出

- 改动 diff 摘要
- 涉及文件列表
- A6 ValidationReport

## 触发方式

- helix 编排，A4 方案确认后自动执行
- `/a5-executor` 手动触发（需先有 PlanDoc）
